Database Schema Design

people
  • id (PK)
  • user_id (FK)
  • first_name
  • last_name
  • gender
  • date_of_birth
relationships
  • id (PK)
  • person_id (FK)
  • relative_id (FK)
  • relationship_type
  • is_ex
media
  • id (PK)
  • attachable_id (FK)
  • attachable_type
  • title
  • description
